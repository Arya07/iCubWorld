Welcome to iCubWorld!

How many objects can iCub recognize? This question led us in 2013 to start the iCubWorld project, with the main goal of benchmarking the development of the visual recognition capabilities of the iCub robot. The project is the result of a collaboration between the Istituto Italiano di Tecnologia (IIT) - iCub Facility, the University of Genoa - SlipGURU research group, and the Laboratory for Computational and Statistical Learning.

iCubWorld datasets are collections of images recording the visual experience of iCub while observing objects in its typical environment, a laboratory or an office. The acquisition settings are devised to allow a natural human-robot interaction, where a teacher verbally provides the label of the object of interest and shows it to the robot, by holding it in the hand; the iCub can either track the object while the teacher moves it, or explore it by itself taking it in its hand.

Since 2013, we published four iCubWorld releases of increasing size (described in detail below), aimed at investigating complementary aspects of robotic visual recognition. Indeed, these image collections allow for extensive analysis of the behaviour of recognition systems when trained in different conditions, offering a faithful and reproducible benchmark for the performance that we can expect from the real system.

Acquisition

Images comprising iCubWorld datasets are annotated with the label of the object represented and a bounding box around it. We developed a Human-Robot-Interaction (HRI) application to acquire annotated images by exploiting the real-world robotic context. This schema allows to build large annotated datasets in a fast and natural way.

The only external supervision is in the form of a human teacher providing verbally the label of the object that is going to be acquired. The teacher approaches the robot and shows the object in his/her hand; during the acquisition, localization of the object in the visual field of the robot is performed by exploting self-supervision techniques.

Two acquisition modalities are possible: human or robot mode. In human mode, the teacher moves the object holding it in the hand and the robot tracks it by exploiting either motion or depth cues. In robot mode, the robot takes the object in the hand and focuses on it by using knowledge of its own kinematics. We record incoming frames from the robot cameras, togheter with the information on the approximate bounding box, for a fixed time period.

More detailed information on the acquisition setup can be found in our related publications.

Datasets

iCubWorld is an ongoing project. Latest release is iCubWorld - Transformations.

  • No. objects: 150
  • No. of categories: 15 (10 obj/cat)
  • No. acquisitions per obj: 10 (visual transformations, days)
  • Acquisition mode: human
  • Segmentation cue: depth
  • Whole image available: yes
  • Acquired camera: left, right
This dataset is far larger and more articulated than previous releases: beyond comprising more objects and categories, each object has been acquired while undergoing five visual transformations, in two different days. The main motivation is to provide a sufficiently large benchmark for studying how fine-tuning latest deep Convolutional Neural Networks on typical images acquired in robotics settings, and containing strong visual nuisances, affects the invariance properties gained by training on wide web data collections.
  • No. objects: 28
  • No. of categories: 7 (4 obj/cat)
  • No. acquisitions per obj: 4 (days)
  • Acquisition mode: human
  • Segmentation cue: motion
  • Whole image available: yes
  • Acquired camera: left
The peculiarity of this dataset is that each object has been acquired on four different days: the four acquisitions differ for little changes in the environment (e.g., same room but different light). This allows for studying incremental learning techniques where the knowledge of objects can be refined with experience gained while observing them in different conditions.
  • No. objects: 40
  • No. of categories: 10 (4 obj/cat)
  • No. acquisitions per obj: 1
  • Acquisition mode: human (train), both (test)
  • Segmentation cue: motion
  • Whole image available: no
  • Acquired camera: left
This release has been devised in order to allow for testing generalization properties of recognition methods from one setting to another. Indeed, while the training set has beeen acquired in human mode, the test set is available in both modes and with different human operators.
  • No. objects: 7
  • No. of categories: 7 (1 obj/cat)
  • No. acquisitions per obj: 1
  • Acquisition mode: both
  • Segmentation cue: motion
  • Whole image available: no
  • Acquired camera: left
This is the first release of iCubWorld, where we present first the developed HRI acquisition setting and perform initial benchamrking of the recognition methods employed on the iCub platform at the time.
